# 資料說明（工程交接用）

**資料根目錄**: `data/ESP32_recored/`

---

## 1. 硬體架構

```
ESP32-S3 (FreeRTOS)
  ├── I2C1 @ 400kHz ──┬── ADS1115_1 (0x48) ── 4-ch single-ended
  │                    └── ADS1115_2 (0x49) ── 4-ch single-ended
  │                         × 4 MUX settings = 32 壓力通道
  └── I2C0 ── MAX30105 PPG（指夾式，Green LED）
```

### 壓力感測器（FSR × 32）

- **物理佈局**: 16 列 × 2 行矩形網格，鋪在乳膠床墊上方（論文圖 3-2）
  - 下排標號 1–16（左→右），上排標號 17–32（右→左，反向）
- **感測原理**: FSR 分壓電路。**ADC 值越高 ≈ 越沒壓力，越低 ≈ 壓力越大**
  - 無接觸：FSR 高阻抗 → 分壓節點 ≈ 滿量程 → ADC ≈ 21,800–22,000
  - 有接觸：FSR 低阻抗 → 分壓節點下降 → ADC ≈ 1,900–11,000
  - BCG 訊號 = 疊加在靜態偏壓上的**微小動態波動**（心跳造成的體重位移）
- **ADC**: ADS1115 16-bit，DR=860 SPS，單端量測
  - 錯誤碼：`-2`（讀取失敗）、`-3`（初始化失敗）、`-4`（設備代碼無效）
- **通道編號**（韌體掃描順序）：
  ```
  ch = mux_idx × 8 + adc_idx
  mux_idx ∈ {0,1,2,3}    # MUX 切換 (GPIO35/36)
  adc_idx ∈ {0..7}        # 0–3 → ADS_1, 4–7 → ADS_2
  ```

### PPG 參考（MAX30105）

- 指夾式配戴，Green LED mode，sampleRate=800，pulseWidth=215，adcRange=16384
- 作為心率 Ground Truth，**不是部署時的輸入**

### 取樣率限制

韌體以 FreeRTOS task 依序輪詢 ch0→ch31，每通道 I2C 讀取約 2.28 ms：

| 韌體版本 | 掃描通道 | T_cycle | 取樣率 |
|---------|---------|---------|-------|
| 32ch（101–223）| ch0–31 | ≈ 74,000 µs | **13.5 Hz** |
| 16ch（224_16ch）| ch0–15 | ≈ 37,000 µs | **27.03 Hz** |

> ch0 最先被採樣，ch31 最後；通道間最大延遲 ≈ 0.97 sample（72 ms）。

## 2. 原始檔案格式

每姿勢一個檔案：`ppg_data_YYYYMMDD_HHMMSS.txt`，逗號分隔，無標題行。

| 欄位 | 名稱 | 類型 | 說明 |
|------|------|------|------|
| 0 | timestamp | string | PC 時間 `"YYYY-MM-DD HH:MM:SS.mmm"` |
| 1–32 | ch0–ch31 | int16 | FSR 壓力 ADC（高=無壓，低=有壓）|
| 33 | ppg | int32 | MAX30105 PPG 讀值 |
| 34 | esp32_us | int64 | `esp_timer_get_time()` 微秒時間戳 |

範例：
```
2025-05-20 13:14:43.343,21879,21879,...,5092,3267,...,21888,99840,991080717
```

## 7. 下游處理入口

```
原始 .txt
  ↓ generate_ground_truth.py        → PPG FFT → 每窗口 Ground Truth HR
  ↓ stage1/step1_generate_annotations.py → 通道可靠度標註
  ↓ thesis_unified/common.py: load_all_data() → 載入訓練資料
```

**Ground Truth 參數**：128-pt 窗口 @ 13.5 Hz（9.48 秒）、50% overlap、Hanning FFT、0.75–3.0 Hz 取峰值 × 60 = BPM。相鄰窗口 HR 跳變 > 20 BPM 以前向填充修正。
